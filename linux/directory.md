# Linux Filesystem Directory Structure

|Directory|Purpose|
|---|---|
|`/`|Root directory - the top of the entire filesystem tree|
|`/bin`|Essential command binaries for all users (e.g., ls, cp, cat)|
|`/boot`|Static files for the boot loader (kernel, initramfs, GRUB configs)|
|`/dev`|Device files - interfaces to hardware devices (e.g., /dev/sda, /dev/tty)|
|`/etc`|Host-specific system-wide configuration files|
|`/home`|User home directories (e.g., /home/username)|
|`/lib`|Essential shared libraries needed by /bin and /sbin|
|`/lib64`|64-bit essential shared libraries (on 64-bit systems)|
|`/media`|Mount points for removable media (USB drives, CDs, etc.)|
|`/mnt`|Temporary mount points for filesystems (manual mounting)|
|`/opt`|Optional/add-on software packages (third-party applications)|
|`/proc`|Virtual filesystem - process and kernel information (not real files)|
|`/root`|Home directory for the root (superuser) account|
|`/run`|Runtime data for running processes (replaces /var/run in modern systems)|
|`/sbin`|System administration binaries (requires root privileges)|
|`/srv`|Data for services provided by the system (web, FTP, etc.)|
|`/sys`|Virtual filesystem - device and driver information (sysfs)|
|`/tmp`|Temporary files (often cleared on reboot)|
|`/usr`|User programs, libraries, and documentation (secondary hierarchy)|
|`/usr/bin`|Non-essential user command binaries|
|`/usr/sbin`|Non-essential system administration binaries|
|`/usr/lib`|Libraries for /usr/bin and /usr/sbin|
|`/usr/local`|Locally installed software (manually compiled packages)|
|`/usr/share`|Architecture-independent shared data (docs, icons, etc.)|
|`/var`|Variable data files (logs, databases, caches, mail spools)|
|`/var/log`|System and application log files|
|`/var/cache`|Cached data from applications|
|`/var/spool`|Queued work (print jobs, mail, cron tasks)|

---

## Quick Reference Notes

- **Virtual filesystems**: `/proc`, `/sys`, and sometimes `/dev` are virtual - they don't exist on disk but are generated by the kernel
- **Immutable vs. Variable**: `/etc`, `/bin`, `/lib` rarely change; `/var`, `/tmp`, `/run` change frequently
- **User vs. System**: `/home` is for regular users; `/root` is specifically for the superuser
- **Mount points**: `/mnt` is for temporary manual mounts; `/media` is typically auto-mounted removable media

---

_Note: Some distributions may have slight variations (e.g., systemd-based systems use `/run` instead of `/var/run`)._
